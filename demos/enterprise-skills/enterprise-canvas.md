## 二、核心痛点（请在适用的前打✓，可多选）

### 效率类痛点
- [ ] **A. 重复文档处理**：大量重复性文档处理工作（报告/合同/工单/审批表），人工耗时且易出错
- [ ] **B. 跨部门协同慢**：信息在部门间流转慢，邮件/微信沟通难以追踪
- [ ] **C. 决策周期长**：会议多、流程长，专家意见难以快速汇聚
- [ ] **D. 知识难以复用**：员工经验/规程/历史项目留在个人脑中，查找耗时

### 决策类痛点
- [ ] **E. 数据富洞察穷**：数据有但分析能力弱，决策还是拍脑袋
- [ ] **F. 专家资源稀缺**：领域专家少，评审排期长
- [ ] **G. 风险识别滞后**：问题爆发后才处理，缺乏预警机制

### 创新类痛点
- [ ] **H. 研发/创新慢**：从想法到原型/产品的周期太长
- [ ] **I. 文献/资料消化慢**：技术报告/论文/政策文件多，阅读消化能力瓶颈
- [ ] **J. 遗留系统包袱**：旧系统多，新技术难以融合，改造风险高

### 质量/合规类
- [ ] **K. 质量靠人工抽检**：自动化不足，漏检率高，质量依赖个人经验
- [ ] **L. 合规审查繁重**：合规文件多，人工审查容易遗漏，且占用大量专业人力
- [ ] **M. 测试覆盖率低**：软件测试不充分，Bug修复成本高，发布风险大

## 如何使用本文件

1. 填写完成后，保存为 `enterprise-canvas.json`（可由 `/enterprise-profile` 命令自动生成）
2. 运行 `/map-opportunities` 查看AI机会地图
3. 运行 `/select-demos` 获取推荐Demo组合
4. 运行 `/customize-scenarios` 生成企业专属演示脚本
5. 运行 `/build-narrative` 构建完整4幕叙事
6. 运行 `/generate-package` 生成可执行的RUNBOOK和CrewAI配置
