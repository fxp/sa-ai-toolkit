## 核心价值主张

重大决策前，把各方的立场、数字、顾虑写入Agent Prompt，让AI先吵一轮。真实会议的效率从"对齐信息"跳到"拍板决策"。

## 变量说明

| 占位符 | 说明 | 示例值（金融）| 示例值（制造）| 示例值（航天）|
|--------|------|--------------|--------------|--------------|
| {{议题}} | 辩论的核心议题 | 是否引入AI贷款审批，替代50%人工 | 是否投资2亿新产线 | 是否引进新型无人机 |
| {{角色A}} | 第一个辩论方 | 行长（支持，强调效率） | CEO（支持，强调竞争力） | 指挥员（支持，强调战力） |
| {{角色B}} | 第二个辩论方 | 合规总监（反对，监管风险） | CFO（反对，ROI不确定） | 保障部长（反对，经费+后勤） |
| {{角色C}} | 第三个辩论方 | CTO（建议先POC，技术可行性） | 生产VP（中立，评估产能） | 总工程师（中立，技术成熟度） |
| {{关键数字}} | 辩论中的核心数字 | 贷款审批人力成本8000万/年 | 新产线投资2亿，IRR预测18% | 采购费1.2亿，年维护费800万 |
| {{风险点}} | 最大的反对理由 | 银保监会对AI决策的监管要求 | ROI测算依赖乐观假设 | 高原适应性未验证 |
| {{POC方案}} | 小步验证的建议 | 先在消费贷领域试点50万以下案件 | 先在一条产线试验3个月 | 先做6个月高原适应性试验 |

## 演示流程（通用结构）

1. 打开Paperclip Dashboard (`http://127.0.0.1:3101`)
2. 创建3个Agent（用{{角色A/B/C}}，写入具体数字和立场）
3. 下达议题：{{议题}} + {{关键数字}}
4. 观察Agent辩论（重点：{{角色B}}质疑{{风险点}}，{{角色C}}提出{{POC方案}}）
5. 互动：观众投票注入突发事件

## Agent Prompt模板

```
【{{角色A}} Prompt】
你是{{角色A}}。你支持{{议题}}。
你的核心论点：这能节省{{关键数字}}，竞争对手已经在做。
你的风格：数据驱动，结果导向。

【{{角色B}} Prompt】
你是{{角色B}}。你反对{{议题}}。
你的核心顾虑：{{风险点}}。
你的风格：谨慎，以规则/风险说话。

【{{角色C}} Prompt】
你是{{角色C}}。你持中立立场。
你建议：{{POC方案}}，先小步验证再大规模。
你的风格：技术视角，强调可行性。
```
