# 企业AI培训师使用手册

> 本手册面向：使用本课程给企业做AI培训/演示的培训师
> 唯一入口：`/demo-full-gen {企业名称}`

---

## 一、完整工作流（从接到客户到交付演示）

```
接到客户 ──→ 生成方案 ──→ 审阅定制 ──→ 现场演示 ──→ 跟进落地
  │              │            │            │            │
  │              │            │            │            └── 发送行动建议
  │              │            │            └── 按日程逐Demo演示
  │              │            └── 根据客户反馈修改Demo内容
  │              └── /demo-full-gen {企业名}
  └── 了解客户行业和受众
```

---

## 二、Step by Step 操作指南

### Step 1：一键生成（5分钟）

```bash
# 在Claude Code中输入：
/demo-full-gen 招商银行
```

系统自动完成：
1. 搜索企业信息 → 生成企业画像
2. 识别行业 → 跳过不适用的Demo
3. 逐个Demo填充定制化内容
4. 输出到 `outputs/招商银行/` 目录

**生成后你会得到：**

| 文件 | 作用 | 你需要做什么 |
|------|------|------------|
| `00-企业画像.md` | 企业基本信息 | **审阅**，补充你了解的内部信息 |
| `01-Demo选型评分.md` | Demo推荐排序 | **调整**，根据受众偏好增删Demo |
| `02-演示排序和时间分配.md` | 日程表 | **确认**，调整时间分配 |
| `03-开场台词.md` | 每个Demo的开场白 | **练习**，改成你自己的口吻 |
| `05-行动建议.md` | AI落地路线图 | **定制**，结合客户实际情况 |
| `demos/*.md` | 16-20个Demo定制内容 | **逐个审阅**，修改不准确的细节 |

### Step 2：审阅和定制（30-60分钟）

**必须检查的3件事：**

#### ① 企业画像是否准确

打开 `00-企业画像.md`，检查：
- 产品名称是否正确（不是所有银行都有"金葵花"）
- 竞品列表是否合理
- 痛点是否命中客户真实需求

**如何修改：** 直接编辑.md文件，或告诉Claude：
```
画像中的竞品列表需要更新，招行最近更关注的竞品是浦发银行而不是兴业银行
```

#### ② Demo选型是否合理

打开 `01-Demo选型评分.md`，根据你和客户的沟通调整：

| 情况 | 操作 |
|------|------|
| 客户只有管理层来看 | 保留demo-ceo-board、demo-org-uplift、demo-ppt-gen，删掉技术类Demo |
| 客户是纯技术团队 | 删掉demo-ceo-board，加重demo-kaggle、demo-playwright |
| 客户时间只有1小时 | 只保留评分前5的Demo |
| 客户特别关心某个话题 | 手动提升该Demo的优先级 |

#### ③ 定制内容是否用了真实术语

打开 `demos/` 下的每个文件，检查：
- ❌ "你的产品" → ✅ "招行App"
- ❌ "VIP客户" → ✅ "金葵花客户"
- ❌ "理财产品" → ✅ "招赢睿远稳健3号"

**如何修改：** 直接编辑，或告诉Claude：
```
demo-roleplay.md 中的理财产品名称需要改成"招银理财·招赢稳健"，
客户等级改成"钻石客户"而不是"金葵花"
```

### Step 3：现场演示

#### 演示前准备（提前1天）

```bash
# 1. 确认所有Web Demo可用
#    有Web标记的Demo需要测试URL能否打开

# 2. Org-Uplift Game需要启动本地服务器
cd demos/demo-metr-org-uplift
python3 -m http.server 8765 &

# 3. 准备API Key（如果要现场LLM调用）
export BIGMODEL_API_KEY=your_key_here

# 4. 准备备用录屏（防止网络故障）
```

#### 演示中操作

**培训师的实际操作就两件事：**

1. **读开场台词** — 从 `03-开场台词.md` 中对应Demo的开场白开始
2. **执行Demo步骤** — 打开 `demos/demo-xxx.md`，按里面的Step 1/2/3操作

**具体示例——demo-roleplay（最常用的Demo）：**

```
培训师操作流程：

1. 读开场台词：
   "金葵花客户王总，持有500万招赢睿远，净值回撤15%..."

2. 邀请观众互动：
   "在座哪位愿意来当客服？"

3. 在Claude Code中输入角色定义：
   （从 demos/demo-roleplay.md 中复制IDENTITY.md的内容）

4. 让观众和AI对话（通过Claude Code或飞书）

5. 对话结束后，让AI生成评估报告

6. 展示评分维度和改进建议

7. 过渡到下一个Demo：
   "刚才我们看到AI能当培训师。那如果AI不只是陪练，
    还能帮你做行业研究呢？我们看下一个Demo..."
```

### Step 4：跟进落地

演示结束后，发送 `05-行动建议.md` 给客户，其中包含：
- 本周立即可做的3件事
- 本月部门试点的4件事
- 本季全行推广的4件事
- 关键KPI指标

## 三、常见定制场景

### 场景A：客户说"我们不是银行，是保险公司"

```
/demo-full-gen 中国平安

# 系统会自动：
# - 识别为保险/金融行业
# - 产品名改为：平安福、玺越人生、陆金所
# - 角色扮演场景改为：理赔投诉、保单纠纷
# - 知识库改为：核保规则、理赔标准
```

### 场景B：客户说"我们是制造业，不需要金融Demo"

```
/demo-full-gen 宁德时代

# 系统会自动：
# - 识别为制造/新能源行业
# - 推荐SAM3质检、Ontology、YOLO优化、电路设计
# - 跳过FinRobot行研（或改为产业链分析）
# - 角色扮演改为：供应商审核谈判
```

### 场景C：客户说"我们只要一个Demo，做深一点"

不需要用 `/demo-full-gen`。直接告诉Claude：

```
帮我为招商银行深度定制一个角色扮演Demo：
- 场景：金葵花客户投诉招赢睿远净值亏损
- 要求：5个难度级别，每级有不同的情绪触发条件
- 包含：评分维度、评语模板、改进建议
- 输出：完整的IDENTITY.md可以直接使用
```

### 场景D：客户说"演示内容不够行业化"

打开生成的Demo文件，告诉Claude具体修改：

```
demos/demo-karpathy-kb.md 中的知识库内容太通用了。
招行的信贷审批有以下特殊流程：
1. 闪电贷是纯线上自动审批
2. 经营贷需要经营场景验证
3. 房贷需要对接住建委数据
请基于这些信息重新生成知识库的示例内容。
```

### 场景E：想增加一个全新的Demo

```
帮我为招商银行新增一个Demo：
名称：智能催收Agent
场景：逾期客户分级催收（M1/M2/M3）
要求：
- AI根据客户画像自动选择催收话术
- 合规边界：不能在什么时间段打电话、不能说什么话
- 催收效果跟踪和策略优化
输出到 outputs/招商银行/demos/demo-collection-agent.md
```

## 四、培训师常见问题

### Q: 生成的内容有错误怎么办？

**直接编辑文件**或告诉Claude修改。所有内容都在 `outputs/{企业}/` 目录下，都是纯文本.md文件，用任何编辑器都能改。

### Q: 怎么给不同受众做不同版本？

在 `02-演示排序和时间分配.md` 中已经有120分钟版和60分钟版。你也可以让Claude生成更多版本：

```
基于招商银行的Demo包，生成一个"纯管理层版"30分钟日程，
只保留3个最震撼的Demo，每个10分钟。
```

### Q: 现场网络不好怎么办？

1. **Org-Uplift Game**：提前用一键模拟跑完一次，截图/录屏备用
2. **LLM调用**：不填API Key则用离线骰子模拟
3. **Web Demo**：提前截图放在PPT里作为备份
4. **角色扮演**：不联网时直接用Claude Code本地模式

### Q: 客户想要PPT而不是Markdown？

```
把 outputs/招商银行/ 下的所有内容整合成一份PPT，
用招行品牌色(红#CC0000)，每个Demo一页，
包含：Demo名称、一句话价值、操作步骤、预期效果。
```

这会调用 `/anthropic-skills:pptx` 自动生成。

### Q: 怎么评估培训效果？

在 `05-行动建议.md` 中有KPI指标。培训后1个月跟进：
- 有多少人开始用AI工具？
- 哪个Demo被复用最多？
- 最大的落地障碍是什么？
