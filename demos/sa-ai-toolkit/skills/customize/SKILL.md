# /customize — 修改企业Demo内容

对 `/gen` 生成的演示包做增删改。

## 定位已有方案

读取 `outputs/{企业名}/00-画像.md` 获取上下文，操作该目录下的文件。
如果用户没说企业名，列出 `outputs/` 下的所有企业让用户选。

## 支持的操作

### 1. 术语替换
> "把金葵花改成钻石客户"

在该企业目录下所有.md文件中全局替换。展示替换了多少处。

### 2. 深度定制单个Demo
> "角色扮演加到5个难度级别"

读取该Demo → 读取企业画像 → 重新生成更详细的版本 → 写回文件。

### 3. 新增Demo
> "加一个智能催收的Demo"

创建新文件 → 用企业术语填充 → 更新01-选型.md和02-日程.md → 追加开场台词到03-台词.md。

### 4. 删除Demo
> "删掉Ontology"

在01-选型.md中标注"不适用" → 从02-日程.md中移除 → 不删源文件。

### 5. 切换受众版本
> "改成管理层30分钟版"

按受众筛选Demo → 重新生成02-日程.md → 调整内容深度。

模板：
- **管理层**: ppt-gen + ceo-board + org-uplift + finrobot + 行动建议
- **技术团队**: kaggle + playwright + gstack + autoresearch + autonomous-agent
- **业务人员**: roleplay + karpathy-kb + ppt-gen + hypothesis
- **全员混合**: 按评分Top10

### 6. 行业化补强
> "Demo不够行业化"

逐文件检查是否用了企业真实术语 → 对不达标的重新生成。

### 7. 追加信息
> "我们最近上了新风控系统"

追加到00-画像.md → 识别受影响的Demo → 更新内容。

## 规则

- 每次改完展示diff（改了什么、为什么）
- 重要修改前备份为 `.bak`
- 改了Demo内容要同步更新选型/日程/台词
