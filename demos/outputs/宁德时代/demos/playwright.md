# Playwright: MES/ERP系统Web自动化测试

## Demo 概述

用Playwright + AI为宁德时代的MES(制造执行系统)和ERP系统编写自动化Web测试脚本，覆盖核心业务流程——生产工单下发、物料领用、质检报告录入、产线切换审批。解决灯塔工厂数字化系统频繁升级后的回归测试痛点。

## 测试痛点

宁德时代灯塔工厂的数字化系统现状：
- MES系统每月2-3次迭代升级，涉及6大工序的核心流程
- ERP与MES对接接口众多，升级后回归测试耗时5-7天
- 手动测试覆盖率仅60%，漏测导致产线停机2次/季度
- 测试人力不足——IT测试团队仅20人，要覆盖50+核心业务流程

## 核心测试场景

### 场景1: 生产工单流程
```
测试路径:
  登录MES → 创建生产工单(麒麟电池/神行超充)
  → 选择产线(涂布线#3) → 绑定BOM(200+物料)
  → 下发工单 → 验证产线接收 → 验证物料拉动
```

### 场景2: 质检报告录入
```
测试路径:
  登录QMS → 选择检验批次 → 录入涂布面密度数据
  → 录入卷绕对齐度数据 → 判定OK/NG
  → 验证NG电芯自动隔离 → 验证8D报告触发
```

### 场景3: 产线切换审批
```
测试路径:
  登录MES → 发起产线切换申请(麒麟→神行超充)
  → 填写切换参数(涂布/卷绕/注液参数变更)
  → 提交审批 → 工艺工程师审批 → 质量总监审批
  → 验证参数自动下发到设备PLC
```

### 场景4: 物料追溯查询
```
测试路径:
  登录MES → 输入电芯编号 → 查询完整追溯链
  → 验证物料批次信息 → 验证设备参数记录
  → 验证操作员信息 → 导出追溯报告
```

## AI辅助测试生成

```
Step 1: AI读取MES页面DOM结构(Playwright录制)
Step 2: 用自然语言描述测试场景
        "测试麒麟电池生产工单从创建到产线接收的完整流程"
Step 3: AI自动生成Playwright测试脚本
Step 4: 自动生成测试数据(工单号/物料编码/检验数据)
Step 5: 执行测试并生成报告
Step 6: 失败用例自动截图+定位失败原因
```

## 演示流程

```
Step 1: 展示宁德时代MES系统登录页面(模拟)
Step 2: 用自然语言描述测试需求
Step 3: AI自动生成Playwright测试脚本(展示代码)
Step 4: 执行自动化测试(实时录屏)
Step 5: 展示测试报告——通过/失败/截图/耗时
Step 6: 演示失败用例的自动定位和修复建议
```
