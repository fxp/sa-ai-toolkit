# GStack: BMS数据看板内部工具快速开发

## Demo 概述

用GStack让AI在10分钟内为宁德时代BMS研发团队搭建一个电池数据看板，整合多源数据（测试系统、MES、LIMS），自动生成电压一致性热力图、充放电曲线、异常电芯标记等核心视图。

## 痛点背景

BMS研发工程师每天面对的数据挑战：
- CTP3.0麒麟电池包包含200+串联电芯，每个电芯有电压、温度、内阻数据
- 数据分散在3个系统：电池测试系统(Arbin/BioLogic)、MES产线数据、LIMS实验室数据
- 工程师花30%时间在"找数据+整合数据"而非"分析数据"
- 每次产品切换（麒麟→神行→骁遥），看板需要重新配置

## 看板功能设计

### 核心视图

| 视图 | 数据源 | 功能 |
|------|--------|------|
| 电压一致性热力图 | 测试系统 | 200串电芯电压分布，标记异常cell |
| 充放电曲线对齐 | 测试系统 | 多电芯充放电曲线叠加对比 |
| 温度场分布 | MES+传感器 | PACK内温度梯度可视化 |
| 容量衰减趋势 | LIMS+测试 | 循环寿命预测曲线 |
| 均衡状态监控 | BMS实时数据 | 主动均衡/被动均衡效果追踪 |
| 异常电芯报告 | 多源融合 | 自动识别+标记异常电芯，生成8D初稿 |

### 数据接口

```
数据源1: 测试系统API → 充放电原始数据(V/I/T vs Time)
数据源2: MES REST API → 产线实时数据(工序参数+质量数据)
数据源3: LIMS导出 → 实验室测试结果(EIS/容量/内阻)
```

## 演示流程

```
Step 1: 描述需求——"做一个麒麟电池BMS数据看板"
Step 2: AI自动生成前端页面(React)——电压热力图+充放电曲线
Step 3: 配置数据接口——模拟200串电芯电压数据
Step 4: 添加异常检测逻辑——电压偏差>30mV标红
Step 5: 一键切换产品线——从麒麟切到神行超充参数
Step 6: 部署为内部Web应用
```

## 技术实现

- 前端：React + ECharts（热力图/曲线图）
- 后端：Python FastAPI（数据聚合+异常检测）
- 部署：Docker容器化，接入宁德时代内网
