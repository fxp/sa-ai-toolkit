# Ontology-MFG: 电池产线全链路知识图谱

## Demo 概述

为宁德时代构建从电芯到PACK的全链路制造知识图谱(Ontology)，覆盖电芯→模组→PACK→产线→设备→物料六级实体关系，实现一键追溯、异常根因定位和跨部门知识协同。

## 制造追溯痛点

宁德时代当前制造数据追溯的困难：
- **6大工序、200+设备类型、13000+物料编码** — 数据散落在MES、ERP、QMS、LIMS等5+系统
- **追溯一个缺陷根因** — 质量工程师需要在多个系统间跳转，平均耗时45分钟
- **跨部门协同** — 质量部发现问题→制造部定位设备→供应链追溯物料，信息断层严重
- **极限制造PPB级要求** — 需要从物料批次级追溯到单个电芯级

## 知识图谱本体设计

### 六级实体层次

```
Level 1: 产品
  └── 麒麟电池包 / 神行超充电池包 / 骁遥电池包 / 天恒储能系统

Level 2: PACK
  └── PACK编号 → 模组组成 → CTP结构信息 → 散热方案

Level 3: 模组/电芯
  └── 电芯编号 → 正极/负极/隔膜/电解液 → 化成数据 → 分选等级

Level 4: 产线
  └── 产线编号 → 工序流程 → 产能/节拍 → 班次信息

Level 5: 设备
  └── 设备编号 → 类型(涂布机/卷绕机/注液机...) → 参数 → 维保记录

Level 6: 物料
  └── 物料编码 → 供应商 → 批次 → 质检报告 → 保质期
```

### 关系类型

| 关系 | 说明 | 示例 |
|------|------|------|
| composed_of | 组成关系 | PACK → 包含 → 电芯×200 |
| produced_on | 生产关系 | 电芯 → 生产于 → 涂布机#3 |
| uses_material | 使用物料 | 正极涂布 → 使用 → NCM811批次#2024-0312 |
| tested_by | 检测关系 | 电芯 → 检测于 → AOI设备#7 |
| maintained_at | 维保关系 | 卷绕机#5 → 维保于 → 2025-03-15 |
| supplied_by | 供应关系 | NMP溶剂 → 供应商 → 恩捷股份 |

## 核心应用场景

### 场景1: 缺陷追溯
```
输入: "电芯#20250315-A-00234涂布厚度超标"
追溯路径:
  电芯#234 → 涂布机#3(3月15日B班) → 浆料批次#0312
  → 搅拌罐#2(搅拌时间偏短15min) → NMP溶剂批次#0310(粘度偏高)
结论: 根因为NMP溶剂批次#0310粘度偏高导致浆料流变性异常
```

### 场景2: 影响分析
```
输入: "供应商A的NMP溶剂批次#0310可能有质量问题"
影响范围:
  → 使用该批次的搅拌罐: #2, #5
  → 影响的涂布产线: 麒麟线#1, 神行线#3
  → 影响的电芯数量: ~45,000个
  → 已装入PACK: 120个
  → 已出货: 35个(客户: XX车企)
```

### 场景3: 产线优化
```
输入: "哪台涂布机OEE最低？原因是什么？"
分析:
  涂布机#7 OEE=76%(最低)
  → 停机原因: 模头清洗频繁(每4小时)
  → 关联因素: 使用浆料粘度偏高(供应商B)
  → 对比: 同型号涂布机#3使用供应商A浆料，OEE=89%
```
