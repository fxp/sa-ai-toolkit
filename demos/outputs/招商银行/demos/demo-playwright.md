# Playwright自动化测试：招商银行App Web端测试

## 场景说明

使用AI + Playwright自动化测试框架，对招商银行App的Web端（网页版/H5页面）进行自动化回归测试。覆盖招行核心业务流程，确保每次版本迭代的质量稳定性。

## 测试范围

**招行App Web端核心功能：**
- 登录/身份验证流程
- 账户总览和余额查询
- 朝朝宝申购/赎回流程
- 招赢睿远理财产品浏览和购买流程
- 信用卡账单查询和还款
- 转账汇款流程
- 掌上生活优惠活动页面

## Demo流程

### Step 1: AI生成测试用例
- 输入招行App的核心业务流程描述
- AI自动生成Playwright测试脚本
- 覆盖正向流程和异常场景

### Step 2: 测试脚本示例

**用例1：朝朝宝申购流程测试**
```
1. 打开招行App Web端登录页
2. 输入测试账号凭证，完成登录
3. 导航到"我的理财" → "朝朝宝"
4. 点击"转入"，输入金额1000元
5. 验证确认页面显示正确的利率和到账信息
6. 确认交易，验证成功提示
7. 验证账户余额变化
```

**用例2：信用卡还款流程测试**
```
1. 登录后导航到"信用卡"模块
2. 查看当期账单金额
3. 选择"一键还款"
4. 验证还款金额和扣款账户
5. 确认还款，验证交易状态
```

### Step 3: AI辅助调试
- 测试失败时，AI自动分析失败原因
- 定位是前端代码问题还是后端接口问题
- 自动生成Bug Report并关联JIRA

### Step 4: 测试报告生成
- 自动汇总测试通过率
- 截图记录关键步骤
- 性能数据（页面加载时间、接口响应时间）

## 预期输出

1. **测试套件**：覆盖7大核心模块的Playwright测试脚本
2. **测试报告**：通过率、失败用例详情、截图证据
3. **性能报告**：各页面加载时间基线
4. **Bug清单**：自动发现的UI和功能问题
