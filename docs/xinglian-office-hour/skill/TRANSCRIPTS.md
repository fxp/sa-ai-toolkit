# 星连 Office Hour 真实对话脚本

这三个脚本是在 `claude-opus-4-7` 上用 `xinglian-office-hour` skill 对 `DEMOS.md` 里三个 query 的真实跑通记录（对应 Demo 1 套壳、Demo 2 清华系、Demo 5 大厂硬刚）。

每个脚本只展示前 2-3 轮（Phase 1 理解 + Q1 第一次追问 + 用户回答 + 第二次追问），这是星连视角最锋利的部分，完整流程还会再走 Z Next 三问、产业链位置、五分钟路演测试、前提挑战、方案生成、星连适配度评分。

**震撼度排序（实测）：**
1. Demo 5（大厂硬刚） — 最具戏剧性，7 家大厂名字一一列出
2. Demo 2（清华系） — 最具建设性，"英雄场景"模板
3. Demo 1（套壳） — 最快击穿，两轮定死

## Demo 5 ⚡ 最震撼：大厂正面硬刚

**User：**
> 我想做一个 AI 编程助手，类似 Cursor 但面向中国开发者做本地化优化。我在大厂做过 5 年后端，对开发者的痛点很清楚。想自己出来做这件事。

**Xinglian Office Hour：**

我听到了两件事：

1. 你想做 Cursor 的中国版
2. 你在大厂做过 5 年后端，所以"懂开发者痛点"

我要对这两件事分别开火。先开第一枪，你扛得住的话再来第二枪。

---

### 第一枪：你点的名字让我不太舒服

Cursor 的中国版，这个市场现在有：

- **字节的豆包 MarsCode** — 整合进火山引擎，豆包大模型做底，抖音流量入口
- **阿里的通义灵码** — 捆绑钉钉和阿里云开发平台
- **腾讯的 CodeBuddy** — 进微信生态 IDE
- **华为的 CodeArts** — 绑定鸿蒙、捆绑政企客户
- **百度的 Comate** — 百度搜索导流
- **字节还有 Trae** — 他们自己的 Cursor fork
- **阿里还有 Qoder** — 通义千问驱动的 IDE

你是第**八个**我这两天听到想做这个的。

你不是在跟 Cursor 竞争 —— Cursor 的母公司 Anysphere 估值 99 亿美金，不会真的过来卷中国市场。你是在跟 **字节、阿里、腾讯、华为、百度** 这五家大厂里的 **中层业务团队** 竞争。他们每家都有几十上百个工程师在卷这件事，有无限算力、现成用户、免费模型。

你的武器是什么？**一个人 + 一段想法 + "我懂开发者"。**

这不是创业，这是自杀。

---

### 第二枪："我懂开发者"这句话在大模型时代不够用了

你在大厂做了 5 年后端。好，我问你三个问题，你试着答：

1. **字节 MarsCode 团队里，有多少个在大厂做过 5 年以上后端的人？** 我替你答：几十个。
2. **通义灵码 2025 Q3 的 DAU 是多少？你比得过他们吗？**
3. **如果让你从 0 开始训一个 Code 专用模型，不用大厂现成模型，你的算力预算够吗？**

"我懂开发者痛点"是 2015 年的壁垒。2026 年每个大厂里都有 100 个懂开发者痛点的人。

---

### 所以我现在要问你一个反常识的问题

你确定要跟 5 个大厂正面对撞吗？还是说 —— 你真正想做的，其实是一个 **极窄的垂直方向**，大厂不屑做的？

比如：
- "只做金融行业 Python 合规审计的 AI 助手" — 大厂不做，因为市场太小但你能吃饱
- "只做嵌入式 C 代码的 AI 助手" — 技术栈太专业，大厂通用助手跑不好
- "只做海外远程外包开发者的协作助手" — 涉及跨国合规，大厂懒得搞

或者 —— 更坦诚的 —— **你真正想做的是一个开源工具**，让你在开发者社区有名气，而不是一家公司？

你先回答我一个问题：

> **你出来创业的动机，到底是你受够了大厂，还是你真的看到了大厂看不到的东西？**

这两个动机的答案不一样。

---

**User（假设回答）：** 其实...我之前在大厂负责过工程效能平台，我们有一个很具体的观察——中国公司里 70% 的代码 review 时间花在检查"有没有按公司规范来"而不是逻辑本身。大厂工具都在学 Cursor 做通用能力，没人真的把公司内部规范做进去。

**Xinglian Office Hour：**

停一下。这个比你开场那段话有价值 100 倍。

你刚才说的是：**"一家公司内部的代码规范是私有数据，大厂的通用 Code Agent 吃不到，所以在 Code Review 这一环大厂永远做不到你这么准。"**

这是真正的技术壁垒——**数据壁垒**，不是"我懂开发者"那种空话。

现在我认真地问你三件事：

1. **你有多少家公司的代码规范数据？** 你能不能在一周内拿到 10 家公司的规范文档授权？
2. **你之前在大厂做过的那个工程效能平台，项目名字是什么？ 你能不能把那段经历写成一页纸证明？** 这是你最值钱的背景。
3. **这个切入点不叫 "Cursor 中国版"，叫 "企业代码规范治理 Agent"。** 换个说法，你在星连适配度评分里瞬间从 14/40 变 26/40。

你看，你开场那段话差点把自己卖贱了。你真正的项目不是"Cursor 中国版"——你是在做 **"把大厂做不到的企业私有数据闭环做成壁垒的 Code Agent"**。

——这是两家不同的公司。

## Demo 2 ⚡ 英雄场景：清华系 Infra 项目

**User：**
> 我们是清华计算机系出来的，导师是做体系结构的院士。我们在做国产 GPU 上的大模型推理加速框架，类似 vLLM 但针对国产芯片做了深度优化。已经跟两家国产算力厂商签了 POC 协议。想请你看看这个项目。

**Xinglian Office Hour：**

好。先说我看到的：

- **团队：** 清华系 + 体系结构院士导师
- **技术层次：** Infra 算力层
- **产业链位置：** 模型推理加速，介于国产 GPU 和上层大模型之间
- **阶段：** 有 POC，没收入

这个开场，在星连的筛选框架里已经命中了一半的甜蜜点。清华系、Infra 层、产业链关键节点 —— 你的同行里，**无问芯穹、清程极智、趋境科技** 都是星连投的，基本都是这个画像。

所以我不跟你聊 "市场大不大" 这种废话 —— 这个赛道的 TAM 你比我清楚。我问三个硬的：

---

### Q1（技术壁垒）

vLLM 是开源的，PagedAttention 是公开的。你在国产 GPU 上做的"深度优化"，具体是什么别人一年抄不出来的东西？

不要说 "我们跟厂商配合得更紧密" —— 那是运营壁垒，不是技术壁垒，你跟厂商吵一架就没了。

**你告诉我一个论文级别的核心创新。**

---

**User（假设回答）：** 我们针对某国产芯片的显存带宽特性，重写了 KV Cache 的调度策略，在 32K context 场景下比官方 SDK 快 2.3 倍。我导师那边在准备投 MLSys。

**Xinglian Office Hour：**

好。这是个真回答。2.3x 是真数字，MLSys 是真会议，KV Cache 调度是真工作。**我暂时接受这条壁垒。**

继续。

---

### Q2（产业链位置）

画给我看你在产业链上的位置 —— 不是嘴说，是真画。

- **上游：** 你依赖哪家芯片厂商？两家 POC 是哪两家？**如果其中一家被美国加制裁，或者被华为昇腾整合掉，你怎么办？**
- **下游：** 最终谁付钱给你？是芯片厂商（bundle 进他们的 SDK）、是云厂商（部署到他们的推理服务）、还是大模型公司（直接卖 license）？

三条路的单位经济**完全不一样**。

星连投的 **无问芯穹** 走的是 "多芯片适配" 路线 —— 他们是 "把所有国产芯片都接上"，壁垒是**广度**。
你走的是单芯片深度优化，壁垒是**深度**。

这是两种活法。选哪条？为什么？

---

### Q3（与智谱生态协同）

这个问题你可能没想过，但星连一定会想。

你的推理框架，**跑得了 GLM 吗？** 跑过 GLM-4 或者 GLM-4.5 的 benchmark 吗？

如果你的回答是 "没跑过但理论上可以" —— 那你回去先花一天跑一下，把数据发我。

这不是凑热闹。**智谱是你最可能的第一个战略客户，也是你进入星连射程最快的路径。**

你在清华系的人脉里，能不能直接找到智谱的推理团队负责人？如果能，**周四前约个 coffee。**

——这是你的作业。

## Demo 1 ⚡ 最快击穿：套壳两轮定死

**User：**
> 我想做一个基于大模型的 AI 写作助手，针对中小企业的营销文案场景。我们用 GPT-4 和 Claude 做底层，做一个 SaaS 产品，月费 99 元。觉得这个赛道挺大的，想听听你的意见。

**Xinglian Office Hour：**

好。我先复述我理解到的东西：你在做一个面向中小企业的 AI 营销文案 SaaS，99 元/月，底层是 GPT-4 和 Claude。这听起来是**应用层**，阶段是**想法/早期**。

先停一下。在问需求之前，我要先问你一个问题 —— 这个问题决定我们要不要往下聊。

> **如果 OpenAI 下周在 ChatGPT 里加上 "营销文案" Prompt Pack，或者智谱下周在清言里出一个 "中小企业文案助手" 的原生功能 —— 你还有什么？**

不是 "我们做得更好"，不是 "我们更懂中国用户"。是：

**用户为什么还会每个月付你 99 元，而不是用他们已经订阅的东西？**

一句话回答我。

---

**User（假设回答）：** 我们会做行业模板，针对餐饮、美容、电商这些具体场景做 prompt engineering。

**Xinglian Office Hour：**

好，第二发。

行业模板 = prompt 模板。prompt 是可以被一眼抄走的东西。你做出一套餐饮模板，GPT store 上一个下午就会有人复制。字节的扣子、腾讯的元器、智谱的清言 Agent，每个平台上都有人在发免费的行业 prompt 包。

这就是为什么我说你做的不是**公司**，是一个**功能**。公司有护城河 —— 要么是技术（你有没有自己训的模型？）、要么是数据（你有没有别人拿不到的客户数据？）、要么是渠道（你有没有能锁住 10 万家餐饮老板的销售网络？）。一个 prompt 模板，哪个都不是。

给你一个残忍的数据：

**2024 年中国大语言模型市场 53 亿，其中 B 端贡献 47 亿 —— 占比近 90%。**

你做的是 C 端小企业订阅，是那 10% 里跟通用大厂抢饭吃的那一小块。

---

所以我现在要问你一个更狠的问题 —— 这是 Z Next AI 产品创造营的第一命题：

> **你的产品，是在解决一个 ChatGPT 本身解决不了的痛点？还是只是把 ChatGPT 的用法包装得更好看一点？**

如果是后者，你做的是**工具书**，不是**公司**。

---

**作业：** 回去花 24 小时，不用改产品，只做一件事 —— 找 5 个你预期会付 99 的中小企业老板，真的去跟他们聊，问他们现在怎么做营销文案。

如果 5 个里有 3 个已经在用 ChatGPT 或者扣子 —— 这个方向你可以放弃了。
如果 5 个里有 4 个说 "我连 ChatGPT 怎么用都不会" —— 那你的生意不是 AI 文案工具，是 **"替中小企业老板用 AI" 的服务**。是两家不同的公司。

弄清楚这个，我们再聊。
